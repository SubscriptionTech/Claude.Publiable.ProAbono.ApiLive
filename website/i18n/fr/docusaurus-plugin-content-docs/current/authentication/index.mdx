L'API Live ProAbono utilise **HTTP Basic Auth** — un Agent Key et un API Key encodés ensemble dans l'en-tête `Authorization`.

## Obtenir vos identifiants

1. Connectez-vous au backoffice ProAbono.
2. Accédez à la section **Integration**.
3. Copiez l'**Agent Key** et l'**API Key**.

:::warning
Traitez ces clés comme des mots de passe. Stockez-les dans des variables d'environnement ou un gestionnaire de secrets — ne les commitez jamais dans le code source.
:::

## URL de base

L'URL de base est spécifique à votre Business. `IdBusiness` est l'identifiant numérique interne visible dans la section Integration du backoffice.

| Environnement | URL de base |
|---------------|-------------|
| Production | `https://api-<IdBusiness>.proabono.com/` |
| Pré-production | `https://pp-api-<IdBusiness>.proabono.com/` |
| Dev | `https://api-<IdBusiness>.shop.chezmose.com/` |

## L'en-tête Authorization

Encodez vos identifiants comme suit :

```
Authorization: Basic <base64(AgentKey:ApiKey)>
```

<Tabs>
<TabItem value="curl" label="curl" default>

Le flag `-u` encode les identifiants automatiquement :

```bash
curl -u "<AgentKey>:<ApiKey>" \
  "https://api-<IdBusiness>.proabono.com/v1/Customer/1"
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js
const credentials = btoa(`${process.env.PROABONO_AGENT_KEY}:${process.env.PROABONO_API_KEY}`);

const response = await fetch('https://api-<IdBusiness>.proabono.com/v1/Customer/1', {
  headers: { Authorization: `Basic ${credentials}` },
});
```

:::note
`btoa` est disponible dans le navigateur et Node.js ≥ 16. Pour les versions antérieures de Node.js, utilisez
`Buffer.from(agentKey + ':' + apiKey).toString('base64')` à la place.
:::

</TabItem>
</Tabs>

:::note
Toutes les requêtes et réponses utilisent `Content-Type: application/json`.
:::

## Codes d'erreur liés à l'authentification

| Statut | Signification |
|--------|---------------|
| `401 Unauthorized` | Identifiants manquants ou invalides |
| `403 Forbidden` | Identifiants valides mais action non autorisée |

Consultez [Conventions](../concepts/conventions#http-status-codes) pour la liste complète des codes de statut HTTP.
