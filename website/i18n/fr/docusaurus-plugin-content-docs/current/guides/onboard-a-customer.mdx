Lorsqu'un utilisateur s'inscrit dans votre application, deux appels API à ProAbono suffisent pour créer l'enregistrement client et démarrer son abonnement.

## Prérequis

- Une Offre active dans le backoffice (vous avez besoin de son `ReferenceOffer`)
- Identifiants API (voir [Authentification](../authentication/))

## Étape 1 — Créez le client

`POST /v1/Customer`

```bash
curl -X POST "https://api-<IdBusiness>.proabono.com/v1/Customer" \
  -u "<AgentKey>:<ApiKey>" \
  -H "Content-Type: application/json" \
  -d '{
    "ReferenceCustomer": "user-42",
    "Email": "bob@example.com",
    "Name": "Bob"
  }'
```

Corps de la requête :
```json
{
  "ReferenceCustomer": "user-42",
  "Email": "bob@example.com",
  "Name": "Bob"
}
```

Réponse — `201 Created` :
```json
{
  "Id": 99,
  "ReferenceCustomer": "user-42",
  "Email": "bob@example.com",
  "Name": "Bob",
  "Status": "Enabled",
  "Links": [{ "rel": "self", "href": "/v1/Customer/99" }]
}
```

:::note
L'`Id` retourné ici est l'identifiant interne de ProAbono — vous pouvez toujours utiliser `ReferenceCustomer` dans les appels suivants à la place.
:::

Consultez [Clients](../api-reference/customer/) pour le schéma complet de l'objet et tous les champs disponibles.

## Étape 2 — Souscrivez le client

`POST /v1/Subscription`

```bash
curl -X POST "https://api-<IdBusiness>.proabono.com/v1/Subscription" \
  -u "<AgentKey>:<ApiKey>" \
  -H "Content-Type: application/json" \
  -d '{
    "ReferenceCustomer": "user-42",
    "ReferenceOffer": "plan-starter"
  }'
```

Corps de la requête :
```json
{
  "ReferenceCustomer": "user-42",
  "ReferenceOffer": "plan-starter"
}
```

Réponse — `201 Created` avec le nouvel objet Abonnement.

Consultez [Abonnements](../api-reference/subscription/) pour tous les champs et options de démarrage.
