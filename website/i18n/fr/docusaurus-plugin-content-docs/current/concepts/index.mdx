ProAbono est une plateforme de facturation par abonnement. Les entités suivantes sont les briques de base de toute intégration.

## Hiérarchie des entités

Un **Business** commercialise un produit. Il est subdivisé en un ou plusieurs **Segments** (par région, type de client, devise, etc.). Chaque **Segment** possède un catalogue d'**Offres**. Un **Client** souscrit à une **Offre**, ce qui crée un **Abonnement**. Les abonnements génèrent des **Factures** et des **Écritures de solde** lors de la facturation. Les **Fonctionnalités** définissent ce à quoi un abonnement donne accès ; l'**Usage** reflète l'état actuel de cet accès pour un client. Avant d'effectuer un changement de facturation, un développeur peut demander une **Simulation tarifaire** pour connaître le coût exact.

## Glossaire des entités

| Entité | Description | Ressource |
|--------|-------------|-----------|
| Business | Un tenant — représente un produit par abonnement vendu via ProAbono. Les données sont entièrement partitionnées par Business. | (ressource API non publique) |
| Segment | Une subdivision d'un Business avec sa propre devise, ses règles de facturation et son catalogue d'offres. | (ressource API non publique) |
| Client | Un client final — destinataire et/ou acheteur d'un abonnement. | [Clients](../api-reference/customer/) |
| Offre | Un plan tarifaire. Souscrire un Client à une Offre crée un Abonnement. « Plan » et « Offre » sont synonymes. | [Offres](../api-reference/offer/) |
| Fonctionnalité | Un composant d'une Offre : **on/off** (accès binaire), **limitation** (quota non réinitialisable), ou **consommation** (quota réinitialisé à chaque renouvellement). | [Fonctionnalités](../api-reference/feature/) |
| Abonnement | Relie une Offre à au plus deux Clients : un **destinataire** (qui consomme le service) et un **acheteur** (qui est facturé). | [Abonnements](../api-reference/subscription/) |
| Facture | Un document de facturation. Seul l'acheteur d'un abonnement a des factures associées. | [Factures](../api-reference/invoice/) |
| Écriture de solde | Un crédit ou débit dans l'historique de facturation du client. | [Écritures de solde](../api-reference/balance-line/) |
| Usage | Un instantané des Fonctionnalités auxquelles un Client peut accéder et en quelle quantité. | [Usage](../api-reference/usage/) |
| Simulation tarifaire | Une estimation du coût d'une opération de facturation (abonnement, mise à niveau, etc.) calculée avant son exécution. | [Simulation tarifaire](../api-reference/quoting/) |

## Identifiants partagés

Les développeurs peuvent assigner leurs propres identifiants pour lier les ressources ProAbono à leurs contreparties dans leur propre système (ex. `ReferenceCustomer=user-42`). Si aucun n'est fourni, ProAbono en génère un automatiquement.

| Ressource | Champ | Utilisation typique |
|-----------|-------|---------------------|
| Client | `ReferenceCustomer` | Associé à l'utilisateur ou au compte dans l'application intégrante |
| Fonctionnalité | `ReferenceFeature` | Associé à une fonctionnalité ou capacité protégée dans l'application |
| Offre | `ReferenceOffer` | Utilisé pour démarrer un abonnement de bienvenue/essai à l'inscription, ou pour l'analytique |
| Segment | `ReferenceSegment` | Utilisé pour partitionner les clients par type, région, etc. |

:::tip
Une fois défini, un identifiant partagé peut être utilisé à la place de l'`Id` interne dans tout appel API ciblant cette ressource.
:::
