# Navigation — Technical

Docusaurus-specific sidebar configuration for the ProAbono API Live documentation website.

For the section structure and page hierarchy, see the [functional specifications](../functional/index.md).

---

## Section ordering — _category_.json

Each authored section folder contains a `_category_.json` that sets the sidebar label and position:

```json
{ "label": "<Label>", "position": <n> }
```

Place this file at the root of each section folder (e.g. `website/docs/introduction/_category_.json`).

---

## sidebars.js configuration

Each authored section uses an `autogenerated` entry keyed to its folder. The API Reference section is handled separately: the plugin generates `docs/api-reference/sidebar.ts` (see [content-pipeline.md](content-pipeline.md)), which is imported and placed as the last item.

```js
// website/sidebars.js
import apiSidebar from './docs/api-reference/sidebar.ts';

/** @type {import('@docusaurus/plugin-content-docs').SidebarsConfig} */
const sidebars = {
  docsSidebar: [
    {type: 'autogenerated', dirName: 'introduction'},
    {type: 'autogenerated', dirName: 'getting-started'},
    {type: 'autogenerated', dirName: 'authentication'},
    {type: 'autogenerated', dirName: 'concepts'},
    ...apiSidebar,
    {type: 'autogenerated', dirName: 'guides'},
  ],
};

export default sidebars;
```

---

## Page ordering within sections

Page order within authored sections comes from the `sidebar_position` frontmatter field, or alphabetical filename order as the default.
