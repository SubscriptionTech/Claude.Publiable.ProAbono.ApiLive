# Navigation

Sidebar structure and page hierarchy for the ProAbono API Live documentation website.

## Sidebar sections

The site has five top-level sections, each corresponding to a subfolder under `website/docs/`. Ordering is set by the `position` field in each `_category_.json`.

| Position | Label | Folder | Content type |
|----------|-------|--------|--------------|
| 1 | Getting Started | `docs/getting-started/` | Authored |
| 2 | Authentication | `docs/authentication/` | Authored |
| 3 | Concepts | `docs/concepts/` | Authored |
| 4 | Guides | `docs/guides/` | Authored |
| 5 | API Reference | `docs/api-reference/` | Auto-generated |

Each section folder contains a `_category_.json`:

```json
{ "label": "<Label>", "position": <n> }
```

## Section page maps

### Getting Started

One page — the section index. No sub-pages.

```
docs/getting-started/
  index.mdx    ← "Quick Start"
```

### Authentication

One page — the section index. No sub-pages.

```
docs/authentication/
  index.mdx    ← "Authentication"
```

### Concepts

Two pages: entity glossary and conventions reference.

```
docs/concepts/
  index.mdx           ← "Core Concepts"
  conventions.mdx     ← "Conventions"
```

### Guides

Index page plus one page per cross-resource flow.

```
docs/guides/
  index.mdx                     ← "Guides"
  onboard-a-customer.mdx        ← "Onboard a customer"
  terminate-and-anonymize.mdx   ← "Terminate & anonymize"
```

### API Reference

Auto-generated by `scripts/extract.mjs`. One subfolder per resource, in the order defined by `shared/ProAbonoLive/specs/resources-index.md`.

```
docs/api-reference/
  _category_.json               ← authored once (label: "API Reference", position: 5)
  customer/                     ← position 1
    _category_.json
    index.mdx
    *.mdx
  offer/                        ← position 2
  feature/                      ← position 3
  subscription/                 ← position 4
  invoice/                      ← position 5
  balance-line/                 ← position 6
  customer-address-billing/     ← position 7
  customer-address-shipping/    ← position 8
  customer-payment-settings/    ← position 9
  usage/                        ← position 10
  quoting/                      ← position 11
```

## Docusaurus sidebar configuration

Use `autogenerated` type. Docusaurus reads `_category_.json` and `position` frontmatter to order sections and pages.

```js
// sidebars.js
/** @type {import('@docusaurus/plugin-content-docs').SidebarsConfig} */
const sidebars = {
  docsSidebar: [{ type: 'autogenerated', dirName: '.' }],
};
module.exports = sidebars;
```

No manual sidebar entries are needed. The section order comes entirely from `_category_.json` position values. Page order within a section comes from the `sidebar_position` frontmatter field (add it to authored pages when explicit ordering matters) or alphabetical filename order as the default.
