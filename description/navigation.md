# Navigation

Sidebar structure and page hierarchy for the ProAbono API Live documentation website.

## Sidebar sections

The site has five top-level sections, each corresponding to a subfolder under `website/docs/`. Ordering is set by the `position` field in each `_category_.json`.

| Position | Label | Folder | Content type |
|----------|-------|--------|--------------|
| 1 | Introduction | `docs/introduction/` | Authored |
| 2 | Getting Started | `docs/getting-started/` | Authored |
| 3 | Authentication | `docs/authentication/` | Authored |
| 4 | Concepts | `docs/concepts/` | Authored |
| 5 | Guides | `docs/guides/` | Authored |
| 6 | API Reference | `docs/api-reference/` | Auto-generated |

Each section folder contains a `_category_.json`:

```json
{ "label": "<Label>", "position": <n> }
```

## Section page maps

### Introduction

One page — the section index. No sub-pages.

```
docs/introduction/
  index.mdx    ← "Introduction"
```

### Getting Started

One page — the section index. No sub-pages.

```
docs/getting-started/
  index.mdx    ← "Quick Start"
```

### Authentication

One page — the section index. No sub-pages.

```
docs/authentication/
  index.mdx    ← "Authentication"
```

### Concepts

Two pages: entity glossary and conventions reference.

```
docs/concepts/
  index.mdx           ← "Core Concepts"
  conventions.mdx     ← "Conventions"
```

### Guides

One page per cross-resource action marked `Documented: yes` in `shared/ProAbonoLive/specs/actions.md`. Currently empty — no actions are marked for documentation yet. See the [backlog](../backlog/add-guides.md).

```
docs/guides/
  (no pages yet)
```

### API Reference

Auto-generated by `docusaurus-plugin-openapi-docs` from the OpenAPI spec. The plugin writes **flat MDX files** directly into `docs/api-reference/` — one file per operation — with no subfolders per resource. This is a constraint enforced by the plugin; do not attempt to reorganize the output into subfolders.

Resource grouping in the sidebar is handled entirely by the plugin's `groupPathsBy: 'tag'` option, which reads the `tags` field from each operation in the spec. The plugin also generates `sidebar.ts` (the sidebar slice imported in `sidebars.js`).

```
docs/api-reference/
  proabono-api-live.info.mdx    ← API overview page (generated)
  sidebar.ts                    ← sidebar slice (generated, imported in sidebars.js)
  <verb>-a-<resource>.api.mdx   ← one file per operation (generated)
  *.StatusCodes.json            ← supporting data files (generated)
  *.RequestSchema.json
  *.ParamsDetails.json
```

Do not edit any file in this folder — all content is regenerated on the next `npm run gen-api-docs` run. To change API reference content, update the OpenAPI spec at `shared/ProAbonoLive/shared-specs/pa-live-openapi-3.0.3.yaml`.

## Docusaurus sidebar configuration

Each authored section uses an `autogenerated` entry keyed to its folder. The API Reference section is handled separately: the plugin generates `docs/api-reference/sidebar.ts`, which is imported and placed as the last item.

```js
// sidebars.js
import apiSidebar from './docs/api-reference/sidebar.ts';

/** @type {import('@docusaurus/plugin-content-docs').SidebarsConfig} */
const sidebars = {
  docsSidebar: [
    {type: 'autogenerated', dirName: 'introduction'},
    {type: 'autogenerated', dirName: 'getting-started'},
    {type: 'autogenerated', dirName: 'authentication'},
    {type: 'autogenerated', dirName: 'concepts'},
    {type: 'autogenerated', dirName: 'guides'},
    {
      type: 'category',
      label: 'API Reference',
      link: {type: 'generated-index', title: 'API Reference'},
      items: apiSidebar,
    },
  ],
};

export default sidebars;
```

Page order within authored sections comes from the `sidebar_position` frontmatter field, or alphabetical filename order as the default.
