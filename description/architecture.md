# Architecture

Project structure and build process for the ProAbono API Live documentation website.

## Repo layout

```
Claude.DocApiLive/
├── shared/
│   └── ProAbonoLive/              ← git submodule — API spec source of truth (not deployed)
│       └── shared-specs/
│           └── pa-live-openapi-3.0.3.yaml  ← input to docusaurus gen-api-docs
├── website/                       ← Docusaurus project
│   ├── docs/
│   │   ├── <page-name>/           ← authored (one folder per page)
│   │   ├── …
│   │   └── api-reference/         ← generated by docusaurus gen-api-docs (committed — do not edit manually)
│   │       └── sidebar.ts         ← generated sidebar slice (imported in sidebars.js)
│   ├── src/
│   │   ├── components/
│   │   └── pages/
│   ├── static/
│   ├── docusaurus.config.js
│   ├── sidebars.js
│   └── package.json
└── description/                   ← specification files
```

## Content origins

| Section | Source | How it enters the site |
|---------|--------|------------------------|
| API reference | `shared/ProAbonoLive/shared-specs/pa-live-openapi-3.0.3.yaml` | Generated by `docusaurus gen-api-docs`, committed to `website/docs/api-reference/` |
| Introduction | Authored in this repo | Written directly to `website/docs/introduction/` |
| Getting started | Authored in this repo | Written directly to `website/docs/getting-started/` |
| Authentication | Authored in this repo | Written directly to `website/docs/authentication/` |
| Concepts | Authored in this repo | Written directly to `website/docs/concepts/` |
| Guides | Authored in this repo | Written directly to `website/docs/guides/` |

## Build process

### Updating API content (local)

Run from the `website/` folder after every submodule update:

```bash
cd website
npm run gen-api-docs
```

Writes to `website/docs/api-reference/`. Commit the result before pushing.

### Development server

```bash
cd website
npm start
```

### Production build

```bash
cd website
npm run build
```

Output: `website/build/` — the directory deployed to Azure Static Web Apps.

### Azure Static Web Apps (CI)

The deployment pipeline runs `npm run build` inside `website/`. The submodule is **not** checked out in CI — extracted files in `website/docs/api-reference/` must already be committed.

| Build setting | Value |
|---------------|-------|
| App location | `website` |
| Build command | `npm run build` |
| Output location | `build` |
