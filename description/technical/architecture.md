# Architecture

Repo layout and build commands for the ProAbono API Live documentation website.

## Repo layout

```
Claude.DocApiLive/
├── shared/
│   └── ProAbonoLive/              ← git submodule — API spec source of truth (not deployed)
│       └── shared-specs/
│           └── pa-live-openapi-3.0.3.yaml  ← input to docusaurus gen-api-docs
├── website/                       ← Docusaurus project
│   ├── docs/
│   │   ├── <page-name>/           ← authored (one folder per page)
│   │   ├── …
│   │   └── api-reference/         ← generated by docusaurus gen-api-docs (committed — do not edit manually)
│   │       └── sidebar.ts         ← generated sidebar slice (imported in sidebars.js)
│   ├── src/
│   │   ├── components/
│   │   └── pages/
│   ├── static/
│   ├── docusaurus.config.js
│   ├── sidebars.js
│   └── package.json
└── description/                   ← specification files
```

## Build commands

### Development server

```bash
cd website
npm start
```

### Production build

```bash
cd website
npm run build
```

Output: `website/build/` — the directory deployed to Azure Static Web Apps.

### Updating API content (local)

Run from the `website/` folder after every submodule update:

```bash
cd website
npm run gen-api-docs
```

Writes to `website/docs/api-reference/`. Commit the result before pushing. See [content-pipeline.md](content-pipeline.md) for the full command and plugin configuration.
