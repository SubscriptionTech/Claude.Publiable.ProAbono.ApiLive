# Implementation instructions — Localization (i18n)

## Functional

The website supports two locales: **English** (`en`) and **French** (`fr`).

- A language switcher dropdown appears in the navbar, letting users switch
  between English and French.
- All locales are URL-prefixed: `/en/docs/…` for English and `/fr/docs/…` for
  French. There is no unprefixed root — the root path `/` redirects to
  `/en/docs/getting-started/`.
- All prose content is available in both languages.

The following elements are **never translated** — they must appear verbatim in
all locales:

| Element | Examples |
|---------|---------|
| API field names | `ReferenceCustomer`, `IdSegment`, `Currency` |
| Enum values | `Enabled`, `Suspended` |
| HTTP methods | `GET`, `POST` |
| Endpoint paths | `/v1/Customer`, `/v1/Subscriptions` |
| JSON code blocks | All `## Sample` sections |
| Action identifiers | `customer.get`, `subscription.create` |

Update `description/functional/navigation.md` to document the locale switcher
navbar item and the root redirect behavior.

## Pipeline

### Translation trigger

Claude translates (or re-translates) affected files immediately, in the same
task, after any of the following events:

- A file in `website/docs/` is created or modified (hand-authored section).
- The `shared/ProAbonoLive` submodule is updated (OpenAPI spec changed).
- The user asks to update the site according to the OpenAPI specs.

### Term mapping

The file `scripts/locales/fr.json` defines custom term mappings
(e.g. `"plan" → "offre"`), tone guidance, and preserve rules for the French
locale. Read this file before translating any content.

### Hand-authored sections

1. Write the English source in `website/docs/<section>/`.
2. Create or update the French translation at the same relative path under
   `website/i18n/fr/docusaurus-plugin-content-docs/current/<section>/`.
3. Structure is identical; only prose is translated.

### API reference (generated files)

`docusaurus-plugin-openapi-docs` generates MDX and JSON files into
`website/docs/api-reference/`. Maintain French counterparts under
`website/i18n/fr/docusaurus-plugin-content-docs/current/api-reference/`.

- **MDX files** (`.api.mdx`, `.tag.mdx`, `.info.mdx`): translate frontmatter
  values and prose; keep the binary `api:` field, component imports, JSX
  structure, and `require()` paths verbatim.
- **JSON data files** (`*.ParamsDetails.json`, `*.StatusCodes.json`,
  `*.RequestSchema.json`): translate `description` string values in
  parameters, responses, and schema properties; keep all other fields (`name`,
  `type`, `format`, `enum`, etc.) verbatim.

When the English source is regenerated after a spec change, re-translate the
affected files.

Update `description/pipeline/authoring.md` and
`description/pipeline/openapi.md` to document the French locale pipeline.

## Technical

### Docusaurus configuration

Add the following `i18n` block to `docusaurus.config.js`:

```js
i18n: {
  defaultLocale: 'en',
  locales: ['en', 'fr'],
  localeConfigs: {
    en: { label: 'English', path: 'en' },
    fr: { label: 'Français', path: 'fr' },
  },
},
```

Setting `path: 'en'` on the default locale forces the `/en/` prefix on
English URLs.

Add a `localeDropdown` item to the navbar in `docusaurus.config.js`.

### Root redirect

Implement a root redirect plugin (`rootRedirectPlugin`) in
`docusaurus.config.js` that registers `src/pages/index.js` at `/`. That
component performs the redirect to `/en/docs/getting-started/` via
`@docusaurus/router`.

### Filesystem layout

```
website/
  docs/                                              ← English source (primary)
  i18n/
    fr/
      docusaurus-plugin-content-docs/
        current/                                     ← French document translations
          api-reference/
          getting-started/
          authentication/
          concepts/
          guides/
      docusaurus-theme-classic/
        navbar.json                                  ← translated UI strings
        footer.json
```

French document files mirror `website/docs/` exactly. A missing file falls
back to the English version.

### Generating UI string skeletons

Run the following command once to create the JSON skeleton files for UI string
translation:

```bash
cd website
npm run write-translations -- --locale fr
```

Fill in the translated values in the generated files under
`website/i18n/fr/docusaurus-theme-classic/` before publishing the French
locale.

### Development server

`npm start` serves only one locale at a time. To serve both locales
simultaneously, open two terminals:

```bash
# Terminal 1 — English
npx docusaurus start --locale en --port 3000

# Terminal 2 — French
npx docusaurus start --locale fr --port 3001
```

Note: the `/en/` URL prefix is absent in dev mode for the default locale.
English pages are served at `/docs/…` in dev, not `/en/docs/…`. This only
affects local development; the prefixed URL structure applies to production
builds.

Update `description/technical/architecture.md` and
`description/technical/content-pipeline.md` to document the i18n configuration
and filesystem layout.
