L'API Live ProAbono vous permet de gérer les abonnements par programmation — créer des clients, démarrer ou mettre à jour des abonnements, vérifier l'accès aux fonctionnalités, et simuler les coûts de facturation avant d'effectuer un changement.

## Prérequis

- Un compte backoffice ProAbono avec un Business actif
- Votre **Agent Key** et votre **API Key** — disponibles dans le backoffice sous **Integration**
- Votre **Business ID** (`IdBusiness`) — également dans la section Integration

## Étape 1 — Identifiez votre URL de base

Chaque Business possède sa propre URL de base :

```
https://api-<IdBusiness>.proabono.com/
```

Pour les tests, utilisez la variante de pré-production :

```
https://pp-api-<IdBusiness>.proabono.com/
```

Consultez [Authentification](../authentication/) pour le tableau complet des environnements.

## Étape 2 — Créez un client

<Tabs>
<TabItem value="curl" label="curl" default>

```bash
curl -X POST "https://api-<IdBusiness>.proabono.com/v1/Customer" \
  -u "<AgentKey>:<ApiKey>" \
  -H "Content-Type: application/json" \
  -d '{
    "ReferenceCustomer": "user-1",
    "Email": "alice@example.com",
    "Name": "Alice"
  }'
```

</TabItem>
<TabItem value="nodejs" label="Node.js">

```js
const credentials = btoa(`${process.env.PROABONO_AGENT_KEY}:${process.env.PROABONO_API_KEY}`);

const response = await fetch('https://api-<IdBusiness>.proabono.com/v1/Customer', {
  method: 'POST',
  headers: {
    Authorization: `Basic ${credentials}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    ReferenceCustomer: 'user-1',
    Email: 'alice@example.com',
    Name: 'Alice',
  }),
});
const customer = await response.json();
```

</TabItem>
</Tabs>

:::tip
`curl -u` encode les identifiants en Base64 automatiquement. Pour les autres clients HTTP, encodez `AgentKey:ApiKey` en Base64 et définissez `Authorization: Basic <encoded>`.
:::

## Étape 3 — Inspectez la réponse

Une requête réussie retourne `201 Created` avec le nouvel objet client :

```json
{
  "Id": 42,
  "ReferenceCustomer": "user-1",
  "Email": "alice@example.com",
  "Name": "Alice",
  "Status": "Enabled",
  "Links": [{ "rel": "self", "href": "/v1/Customer/42" }]
}
```

Consultez [Clients](../api-reference/customer/) pour le schéma complet de l'objet.
